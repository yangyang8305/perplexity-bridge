📚 Perplexity Bridge — 使用说明

Perplexity Bridge 是一个本地反向代理，把 Perplexity.ai 的 Web 会话（Cookie）包装成标准 OpenAI Chat Completions API，让任何支持 OpenAI 格式的客户端（Cursor、Continue、OpenCat 等）直接接入。

⚡ 快速开始

第一步 — 获取 Perplexity Session Token

  1. 用浏览器登录 perplexity.ai
  2. 打开开发者工具 → Application → Cookies → www.perplexity.ai
  3. 找到 Cookie 名 __Secure-next-auth.session-token，复制其 Value
  4. 多账号可用逗号分隔，写入 SESSIONS 环境变量

第二步 — 配置 .env

SESSIONS=eyJhbGc...（你的token）,eyJhbGc...（第二个，可选）
ADDRESS=127.0.0.1:8080
APIKEY=                     # 留空 = 不鉴权；填写则所有请求必须带 Bearer Token
PROXY=                      # 可选，如 http://127.0.0.1:7890
IS_INCOGNITO=true           # 建议开启，不记录对话历史
IS_MAX_SUBSCRIBE=false      # Max 订阅用户设为 true 可解锁更多模型
MAX_CHAT_HISTORY_LENGTH=10000  # 超过此字数自动上传为附件

第三步 — 启动服务

go build -o perplexity-bridge . && ./perplexity-bridge
# 或直接
go run .

启动成功后在浏览器打开 http://127.0.0.1:8080 即可看到本页面。

🔧 高级功能

多 Session 轮询

SESSIONS 用逗号分隔多个 token，系统自动轮询。某个 session 失效后会在下次刷新时自动剔除。

自动刷新 Cookie

每 24 小时自动刷新所有 session token 并写入 sessions.json。Dashboard → Refresh Cookie Now 可立即触发。

长文本自动上传

对话历史超过 MAX_CHAT_HISTORY_LENGTH 字节时，自动以附件方式上传至 Perplexity S3，绕过请求体长度限制。

图片输入

消息中 image_url 类型的内容会自动上传至 Perplexity CDN，模型可直接处理。支持 base64 data-URI 和外链 URL。

Function Calling / Tool Use

支持 OpenAI 格式的 tools 字段。Bridge 会先让模型决策是否需要调用工具，再将结果发回，全程透明。

❓ 常见问题

返回 503 — No SESSIONS configured

环境变量 SESSIONS 未设置或为空。检查 .env 文件是否在工作目录，token 是否正确填写。

返回 401 — Invalid API key

客户端发送的 Bearer Token 与 APIKEY 不匹配。若不需要鉴权，将 APIKEY 留空即可。

Session 失效 / 返回 401 from Perplexity

Session token 已过期（通常因长时间未使用或账号在其他地方登出）。点击 Dashboard → Refresh Cookie Now，或重新从浏览器获取 token 并更新 SESSIONS，再点 Reload Config。

所有重试都失败 / 返回 500

可能是 Perplexity 触发了速率限制（429）或账号异常。检查 token 是否有效，等待几分钟后重试。

模型名不对 / 想用搜索

在模型名末尾加 -search，例如 gpt-5.4-search，即可开启联网搜索。Chat 界面点 Search 按钮效果相同。

⚠️ 本服务依赖 Perplexity Web 非官方接口，可能随 Perplexity 页面更新而失效。如遇大面积故障请检查是否有新版本。
